# Android vs iOS 授權方式完全對比

## 🎯 快速對比

| 功能 | Android | iOS |
|------|:---:|:---:|
| 需要選擇瀏覽器 | ✅ 是 | ❌ 否 |
| 「Open in Browser」對話框 | ✅ 有 | ❌ 沒有 |
| 需要「Always Open」打勾 | ✅ 是（推薦） | ❌ 不需要 |
| 難度程度 | ⭐⭐ 中等 | ⭐ 簡單 |
| 下次授權需要重複操作 | ❌ 不需要（打勾後） | ❌ 不需要 |

## 📱 Android 完整步驟

### 第一次授權（需要打勾）

1. **執行 `/login`**
   ```
   Telegram → /login
   ```

2. **看到「Open in Browser」對話框**
   ```
   ┌────────────────────────────────┐
   │ Open in Browser                │
   │                                │
   │ Do you want to open this link? │
   │                                │
   │ ☐ Always open from            │  ← 一定要勾！
   │   auth.monday.com in browser   │
   │                                │
   │   [Cancel]  [Continue]         │
   └────────────────────────────────┘
   ```

3. **勾選「Always open...」**
   - 點擊方框：`☐` → `☑️`
   - 現在方框會被勾選

4. **點擊「Continue」**
   - 進入 Chrome 開啟授權頁面

5. **看到下載頁面後，啟用桌面模式**
   - Chrome 右上角 `⋮` → 向下滑 → 勾選「桌面版網站」

6. **重新整理**
   - 向下拖動或點擊重新整理 `⟳`
   - 應該看到授權表單

7. **完成授權**
   - 輸入 Monday.com 帳號密碼

### 第二次及以後

因為您已經打勾了「Always open」：

1. **Telegram `/login`**
2. **✨ 自動在 Chrome 中打開**（無需對話框）
3. **直接進入授權流程**
4. **✅ 完成！**

## 🍎 iOS 完整步驟

### 每次授權（都相同，因為 Safari 自動打開）

1. **執行 `/login`**
   ```
   Telegram → /login
   ```

2. **✨ 自動在 Safari 中打開**
   - 無需選擇瀏覽器
   - 無需對話框

3. **看到下載頁面後，啟用桌面模式**
   - Safari 右下角 `↑↓` → 向上滑 → 勾選「以桌面版方式查看網站」

4. **重新整理**
   - 向下拖動頁面進行重新整理
   - 應該看到授權表單

5. **完成授權**
   - 輸入 Monday.com 帳號密碼

### 下次授權

流程完全相同（因為 Safari 總是自動帶您到瀏覽器）

## 🔄 iOS 中修改預設瀏覽器

如果您想讓 iOS 預設使用不同的瀏覽器（而非 Safari）：

1. 打開 iOS 設定應用
2. 向下滑找到「應用程式」或「特定瀏覽器名稱」
3. 選擇「預設瀏覽器應用」
4. 選擇您想要的瀏覽器（例如 Chrome、Firefox 等）
5. ✅ 下次點擊連結時會使用新的預設瀏覽器

**注意：** iOS 14+ 支持此功能。舊版 iOS 只能使用 Safari。

## ⚙️ 技術背景

### 為什麼 Android 有「Open in Browser」對話框？

Android 是開放系統，允許多個瀏覽器共存。當點擊網址時，系統詢問用戶要用哪個應用打開。

### 為什麼 iOS 沒有這個對話框？

iOS 的設計屬於「園牆」模式，預設應用通常由 Apple 控制。Safari 是系統級應用，其他瀏覽器表現上實際上是 Safari 的包裝器。因此 iOS 通常直接使用預設瀏覽器。

### 為什麼需要「Always Open」？

因為 Android 每次都會詢問要用哪個應用打開連結。「Always open」可以讓 Android 記住您的選擇，下次就不用再詢問。

## ✅ 最常見的問題

### Q1：我是 iOS 用戶，為什麼指導頁面提到 Android？

**A：** 指導頁面是通用的，同時適用於 Android 和 iOS 用戶。如果您是 iOS 用戶，可以忽略 Android 特定的步驟（例如「打勾」）。

### Q2：我在 iOS 上看不到「桌面版」選項怎麼辦？

**A：** 
1. 確認您使用的是 Safari（其他瀏覽器可能無此功能）
2. 確認您的 iOS 版本夠新（至少 iOS 13+）
3. 嘗試使用 Chrome for iOS，然後切換回 Safari

### Q3：Android 打勾後仍然看到對話框？

**A：** 
- 檢查您是否正確勾選了方框（應該是 `☑️` 而非 `☐`）
- 確認點擊了「Continue」而非「Cancel」
- 嘗試清除 Chrome 的預設設定並重試（設定 → 應用程式 → Chrome → 清除預設值）

### Q4：我想在 iOS 上使用 Chrome 而非 Safari 怎麼辦？

**A：** 
1. 打開 iOS 設定
2. 找到「Chrome」應用設定
3. 選擇「預設瀏覽器應用」或類似選項
4. iOS 14+ 支持此功能，較舊版本可能無法更改預設

### Q5：我是 Android 用戶但不想打勾怎麼辦？

**A：** 您可以不打勾，但這樣的話：
- 每次都會看到「Open in Browser」對話框
- 需要每次都手動選擇 Chrome
- 不推薦，因為打勾可以節省時間

### Q6：打勾對安全性有影響嗎？

**A：** 沒有。「Always open from auth.monday.com」只是告訴 Android：「下次看到來自 auth.monday.com 的連結，自動在 Chrome 中打開」。這不會造成任何安全問題。
